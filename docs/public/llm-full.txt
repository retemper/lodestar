# Lodestar — Full API Reference for LLMs

## Rule Authoring API

### defineRule<TOptions>(definition)

```ts
import { defineRule } from '@retemper/lodestar';

const myRule = defineRule<{ max: number }>({
  name: 'my-plugin/my-rule',       // {plugin-namespace}/{rule-name}
  description: 'What this rule checks',
  needs: ['fs'],                    // 'fs' | 'graph' | 'ast' | 'config'
  schema: { type: 'object', ... }, // JSON Schema for options validation
  async check(ctx) {
    // ctx.rootDir     — absolute project root
    // ctx.options     — typed TOptions from user config
    // ctx.providers   — data source providers
    // ctx.report({ message, location? }) — report a violation
  },
});
```

### definePlugin(factory)

```ts
import { definePlugin } from '@retemper/lodestar';

const myPlugin = definePlugin(() => ({
  name: 'my-plugin',
  rules: [rule1, rule2],
}));

export { myPlugin };  // named export, NOT default
```

### defineConfig(config)

```ts
import { defineConfig } from '@retemper/lodestar';
import { pluginArchitecture } from '@retemper/lodestar-plugin-architecture';

export default defineConfig({
  plugins: [pluginArchitecture],            // direct imports (preferred)
  rules: {
    'architecture/layers': {
      severity: 'error',                    // 'error' | 'warn' | 'off'
      options: { layers: [{ name: 'domain', path: 'src/domain/**' }] },
    },
  },
  include: ['src/**/*.ts'],                 // optional, default: ['**/*']
  exclude: ['node_modules/**'],             // optional
  baseline: '.lodestar-baseline.json',      // suppress existing violations
});
```

## Providers

### FileSystemProvider (needs: ['fs'])

```ts
ctx.providers.fs.glob(pattern: string): Promise<readonly string[]>
ctx.providers.fs.readFile(path: string): Promise<string>
ctx.providers.fs.exists(path: string): Promise<boolean>
ctx.providers.fs.readJson<T>(path: string): Promise<T>
```

### DependencyGraphProvider (needs: ['graph'])

```ts
ctx.providers.graph.getDependencies(file: string): Promise<readonly string[]>
ctx.providers.graph.getDependents(file: string): Promise<readonly string[]>
ctx.providers.graph.hasCircular(entry: string): Promise<boolean>
ctx.providers.graph.getModuleGraph(): Promise<ModuleGraph>
```

### ASTProvider (needs: ['ast'])

```ts
ctx.providers.ast.getSourceFile(path: string): Promise<unknown>  // SWC Module
ctx.providers.ast.getImports(path: string): Promise<readonly ImportInfo[]>
ctx.providers.ast.getExports(path: string): Promise<readonly ExportInfo[]>
```

ImportInfo: `{ source, specifiers: string[], isTypeOnly, location: { file, line?, column? } }`
ExportInfo: `{ name, isTypeOnly, isDefault, source?: string }` (source present for re-exports)

### ConfigFileProvider (needs: ['config'])

```ts
ctx.providers.config.getPackageJson(dir?): Promise<Record<string, unknown>>
ctx.providers.config.getTsConfig(dir?): Promise<Record<string, unknown>>
ctx.providers.config.getCustomConfig<T>(filename, dir?): Promise<T>
```

## Plugin Reference

### @retemper/lodestar-plugin-architecture

Architecture as Code — layers, module boundaries, and circular dependency detection.

**architecture/layers**
Enforces dependency direction between architectural layers.
```ts
options: {
  layers: [{
    name: string,             // human-readable layer name
    path: string,             // glob pattern matching files in this layer
    canImport?: string[],     // layer names this layer may import from
  }],
  allowTypeOnly?: boolean,    // permit type-only cross-layer imports (default: false)
}
```

**architecture/modules**
Forbids importing internal files of a module — requires barrel (index) imports.
```ts
options: {
  modules: string[],          // module root directories enforcing barrel-only imports
  allow?: string[],           // deep import paths permitted as exceptions
  include?: string[],         // glob for files to check
  exclude?: string[],         // glob for files to skip
}
```

**architecture/no-circular**
Detects circular dependency chains between files.
```ts
options: {
  entries?: string[],         // glob for entry points (default: all)
  ignore?: string[],          // glob for files to skip
  maxDepth?: number,          // max cycle chain length
}
```

**architecture/no-circular-packages**
Detects circular dependencies between workspace packages via package.json.
```ts
options: {
  scope?: string,             // e.g. '@retemper' — auto-detected if omitted
}
```

### @retemper/lodestar-plugin-structure

File system structure enforcement — directory existence, forbidden paths, and paired file validation.

**structure/directory-exists**
Verifies that required directories or files exist. Supports auto-fix.
```ts
options: {
  required: string[],         // glob patterns for paths that must exist
}
```

**structure/no-forbidden-path**
Verifies that forbidden paths do not exist.
```ts
options: {
  patterns: string[],         // glob patterns for paths that must NOT exist
}
```

**structure/paired-files**
Verifies that source files have required companion files. Supports auto-fix.
```ts
options: {
  pairs: [{
    source: string,           // glob pattern matching source files
    required: string,         // template: '{dir}/{name}.spec.ts'
    message?: string,         // custom error message
  }],
}
```

## Reporters

Reporters control how results are displayed. Configure via CLI flags or config file.

### Built-in Reporters
- **console** (default) — human-readable terminal output
- **json** — machine-readable JSON output

### Reporter Plugins
- **@retemper/lodestar-reporter-junit** — JUnit XML format for CI systems
- **@retemper/lodestar-reporter-sarif** — SARIF format for GitHub Code Scanning

```ts
// lodestar.config.ts
export default defineConfig({
  reporters: [
    'console',
    ['@retemper/lodestar-reporter-junit', { outputFile: 'results.xml' }],
  ],
});
```

## CLI

```sh
npx lodestar check [--workspace | --no-workspace] [--reporter console|json]
npx lodestar init
npx lodestar graph [--layers]
npx lodestar impact <file>
npx lodestar watch [--reporter console|json]
```

Exit code 0 = no errors. Exit code 1 = errors found. Warnings don't fail.

## Workspace Mode

Auto-detected when pnpm-workspace.yaml exists. Root config runs globally, per-package `lodestar.config.ts` files run independently (no implicit inheritance). Packages without their own config are skipped.
